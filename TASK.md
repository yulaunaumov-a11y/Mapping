# Тестовое задание: маппинг координат между камерами

## Контекст

На холодильнике в торговом зале установлены три кмеры:

- **door2** — основная камера, смотрит на зону холодильника спереди (анфас)
- **top** — камера сверху, смотрит на холодильник сверху вниз
- **bottom** — камера снизу, смотрит на холодильник снизу вверх

Все камеры дают кадры разрешением **3200 × 1800 px**.

**Бизнес-задача:** определить, где находится заданная точка на изображении с камеры **top** или **bottom** на соответствующем кадре **door2**. 

Собран датасет вручную размеченных соответствий между парами кадров. Ваша задача — построить и оценить модель такого маппинга.

## Структура датасета

```
coord_data/
├── split.json                        # список относительных путей к сессиям train/val
├── train/                            # 56 размеченных сессий для обучения
│   └── camera_door2_<дата>_<время>/
│       ├── door2/
│       │   └── frame_<N>.jpg         # кадр с камеры door2
│       ├── top/
│       ├── bottom/
│       ├── coords_top.json           # разметка соответствий: door2 ↔ top
│       └── coords_bottom.json        # разметка соответствий: door2 ↔ bottom
├── val/                              # 14 размеченных сессий для валидации
│   └── camera_door2_<дата>_<время>/
│       └── … (та же структура)
```

### `split.json`

Каждая строка — **относительный путь** от корня `coord_data` к каталогу сессии:

```json
{
  "train": ["train/camera_door2_2025-11-27_14-26-36", ...],
  "val":   ["val/camera_door2_2025-12-19_10-04-08", ...]
}
```

56 сессий в `train/`, 14 — в `val/`. **Разбиение необходимо соблюдать строго: обучение только на путях из `"train"`, финальная оценка — только на `"val"`.** 

### `coords_top.json` / `coords_bottom.json`

Каждый файл — JSON-массив объектов, где каждый объект описывает пару синхронных кадров:

```json
[
  {
    "file1_path": ".../<сессия>/door2/frame_000636.jpg",
    "file2_path": ".../<сессия>/top/frame_000643.jpg",
    "image1_coordinates": [
      {"number": 1, "x": 1611.9, "y": 1440.2},
      {"number": 2, "x": 1187.5, "y": 1435.5},
      ...
    ],
    "image2_coordinates": [
      {"number": 1, "x": 743.96, "y": 524.59},
      {"number": 2, "x": 920.42, "y": 343.37},
      ...
    ]
  },
  ...
]
```

- `image1_coordinates` — пиксельные координаты точек на кадре **door2**
- `image2_coordinates` — пиксельные координаты тех же точек на кадре **top** или **bottom**
- Точки сопоставляются по полю `"number"`: точка №1 в `image1_coordinates` соответствует точке №1 в `image2_coordinates`
- В каждой паре кадров размечено **17–22** точек соответствия

## Задача

**Маппинг пиксельной координаты из верхней/нижней камеры на вид с door2.**

    Аргументы:
        x, y    -- пиксельная координата на кадре верхней/нижней камеры
        
        source  -- "top" или "bottom" (откуда берётся точка)

    Возвращает:
        (x', y') -- предсказанная пиксельная координата на кадре door2

Выбор подхода свободен, использовать фичи изображения так же можно. 

---

## Метрика качества

Качество измеряется **средним евклидовым расстоянием (MED)** в пикселях на валидационном сплите.

Метрику необходимо привести **отдельно** для пар `top→door2` и `bottom→door2` и предоставить отдельным файлом вместе с результатами.
